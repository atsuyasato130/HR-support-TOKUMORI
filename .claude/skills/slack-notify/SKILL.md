Slackチャンネルにメッセージを送信する。

## 🚨 送信前の必須承認フロー

**Slack送信は取り消せない操作です。以下の順序を必ず守ってください:**

1. ドラフト作成 → 2. ユーザーに提示 → 3. 承認取得 → 4. 送信 → 5. 完了報告

**ユーザーが明示的に「送って」「送信して」「OK」「はい」と承認するまで、絶対に send_slack_message を実行しないこと。**

## 1. 送信先チャンネルの確認

引数からチャンネル名を特定する。不明な場合は必ず確認する。

よく使うチャンネル一覧:
- `#general` — 全体連絡
- `#hr-support` — HR支援チーム
- `#candidates` — 候補者対応
- `#operations` — 業務連絡

## 2. チャンネルID取得（hr-support MCP: get_slack_channels）

チャンネル名からチャンネルIDを取得する。取得失敗時はユーザーに正しいチャンネル名を再確認。

## 3. メッセージドラフトの作成（送信はしない）

メッセージを以下のフォーマットで整形する:

```
[絵文字] [タイトル]

[本文]

[タイムスタンプ / 署名（任意）]
```

## 4. ユーザー承認取得（必須）

ドラフトを以下のテンプレートで提示:

```
━━━━━━━━━━━━━━━━━━━━━━━━━
📬 Slack送信内容の確認

送信先: #<channel_name>  (ID: <channel_id>)
送信者: <投稿者名>

━━━ メッセージ本文 ━━━
<draft_message>
━━━━━━━━━━━━━━━━━━━━━━━━━

この内容で送信してよろしいですか？
  • 送信する → 「OK」「送って」
  • 修正する → 修正内容を指示
  • キャンセル → 「やめる」
```

**ユーザーが「OK」「送信」「送って」等を明示するまで待機する。**

## 5. メッセージ送信（承認後のみ）

承認後に `hr-support MCP: send_slack_message` を実行。

## 6. 完了報告

送信成功後に以下を表示:

```
✅ Slack送信完了
  送信先: #<channel>
  送信時刻: <timestamp>
  メッセージID: <ts>
```

## 失敗時の対処

| エラー | 対処 |
|---|---|
| `channel_not_found` | チャンネル名のtypo確認。`get_slack_channels` で正しい名前を再取得 |
| `not_in_channel` | Botが当該チャンネルに未参加。ユーザーに「#<channel> にBotを招待してください」と案内 |
| `rate_limited` | 1分待機後に再送を提案 |
| `invalid_auth` | SLACK_BOT_TOKEN 失効の可能性。ユーザーに `.env` 確認を依頼 |

## ❌ してはいけないこと

- ユーザーの明示的承認なしに `send_slack_message` を実行すること
- 「ドラフトを作成しました、送信します」と一言で済ませてそのまま送信すること
- 機密情報（APIキー・学生個人情報）を含んだメッセージの送信（個人情報は事前に警告）
