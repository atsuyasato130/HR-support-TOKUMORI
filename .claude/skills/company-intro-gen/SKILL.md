企業紹介文を生成する。対象企業・学生タイプ=$ARGUMENTS

## 1. 学生タイプ確認（必須）

引数にタイプ指定がなければ必ず確認する:
「A/B/C/G/M どのタイプ向けに作成しますか？」

タイプ別訴求軸:
- **A（安定志向）**: 福利厚生・安定・ワークライフバランス・長期キャリア
- **B（成長志向）**: スキルアップ・挑戦機会・裁量・成長速度
- **C（社会貢献）**: 社会的意義・ミッション・社会課題解決
- **G（グローバル）**: 海外展開・英語使用機会・グローバルチーム
- **M（マネジメント）**: リーダーシップ機会・早期昇格・マネジメント経験

## 2. 企業情報取得（hr-support MCP: search_salesforce）

企業名でSalesforce検索し、基本情報を取得する。

**検索ヒットなし** → ユーザーに以下を確認:
- 企業名の表記揺れ（カタカナ/漢字/英数字）
- 新規登録が必要な企業か
- 手動で紹介文情報を提供してもらうか

## 3. 説明会日程取得（重要）

SFの **「説明会日程」フィールド** を必ず参照する（「日程」フィールドではない）。

### 日付フィルタリングルール

取得した説明会日程から、**今日以降の日程のみ**掲載する:
- 今日の日付: 実行時の日付を正確に取得（CLAUDE.mdの `currentDate` 参照）
- 過去日程は除外
- ユーザーが特定日付を指定している場合はそれを優先

## 4. LINE納品フォーマットで生成

以下のフォーマットで出力:

```
【企業名】
📌 おすすめポイント
①（タイプ別訴求軸1）
②（タイプ別訴求軸2）
③（タイプ別訴求軸3）

📅 説明会日程
（説明会日程フィールドの値をそのまま記載・翌日以降のみ）
```

## 5. 表記チェック（必須バリデーション）

- ⚠️ 社名に「学生協賛」が含まれる場合は **「ガクセイ協賛」（カタカナ）** に修正
- 絵文字は指定のもの（📌📅）以外使用しない
- ①②③の丸数字を使用（1,2,3 ではない）

## 6. 完了報告

```
✅ 企業紹介文生成完了
  企業: <company_name>
  タイプ: <type>
  掲載日程数: <N>件（過去日程除外）
```

## 失敗時の対処

| 問題 | 対処 |
|---|---|
| SFに企業が見つからない | 表記揺れ確認（学生協賛→ガクセイ協賛等）／新規登録確認 |
| 説明会日程フィールドが空 | ユーザーに「説明会日程が未登録です。手動で追加しますか？」 |
| 掲載可能な日程がゼロ（全て過去） | ユーザーに「今後の説明会予定がありません」と報告し、別案内を提案 |
| 複数企業ヒット | 候補を提示してユーザー選択 |

## ❌ してはいけないこと

- 「日程」フィールド（別物）を参照すること
- 過去日程を掲載すること
- 「学生協賛」表記のまま出力すること
- タイプ確認を省略してデフォルトで生成すること
