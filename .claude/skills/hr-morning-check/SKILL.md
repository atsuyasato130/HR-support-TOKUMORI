朝のHR業務チェックを実行する。
Gmail・Salesforce・Notionを**並列サブエージェントで同時確認**し、メインコンテキストへの汚染を防ぎながら高速にサマリーを返す。

## 実行方法

3つのサブエージェントを並列起動して結果を統合する。

### サブエージェント1: Gmail確認
```
役割: Gmail未読メールを確認し、対応要否を判定する
ツール: hr-support MCP (read_gmail)
出力形式: 箇条書きで「件名 / 送信者 / 対応要否」のみ返す
制約: 本文詳細は不要。対応必要なもの5件以内に絞る
フォールバック: Gmail接続失敗時は「Gmail確認スキップ(理由)」と記録
```

### サブエージェント2: Salesforce面談確認
```
役割: 本日・明日の面談予定を取得する
ツール: hr-support MCP (get_salesforce_summary)
出力形式: 「日時 / 候補者名 / 種別」の一覧のみ返す
制約: 詳細フィールドは不要。面談一覧のみ
フォールバック: SF接続失敗時は「SF確認スキップ(理由)」と記録
```

### サブエージェント3: Notion議事録確認
```
役割: 昨日追加・更新された議事録を確認する
ツール: hr-support MCP (read_notion_database)
database_id: 2bc48452-bd25-8007-838c-c3c707c953c1
出力形式: 「タイトル / 録画日 / 録画日入力済みか」のみ返す
制約: 内容詳細は不要。直近3件まで
フォールバック: Notion接続失敗時は「Notion確認スキップ(理由)」と記録
```

## 失敗時の対処（個別サブエージェント）

1サブエージェントが失敗しても他2つは実行する。全滅した場合のみ全体停止。

| エラー | 対処 |
|---|---|
| MCP接続失敗（全サービス） | `hr-support MCP` プロセス再起動を案内 |
| Gmail 401 | `.env` の `GMAIL_CREDENTIALS` 確認 |
| SF 認証失敗 | `salesforce-dx` 側のセッション切れ。`sf org login web` を案内 |
| Notion 404 | database_id確認・共有設定チェック |

## 推奨アクション

サマリー出力後、優先度に応じて以下を自動提案:

- 🔴 緊急メールあり → `/email` コマンドを提案
- 本日面談あり → `/interview-schedule` で詳細確認を提案
- Notion録画日未入力 → 「録画日を埋めますか？」と確認
