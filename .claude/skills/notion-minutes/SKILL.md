Notion議事録DBに新しいエントリを追加する。

## 事前確認（必須・空白禁止）

以下の情報を確認する。**不足があれば絶対に進まず、必ず質問する:**

| 項目 | 必須度 | 備考 |
|---|:-:|---|
| **録画日** | 🔴 必須 | 「今日」ではなく録画された実日付。空白禁止 |
| 面談相手 | 🔴 必須 | 候補者名 / 企業名 |
| 面談種別 | 🔴 必須 | IS / FS / 最終 / カジュアル / 商談 等 |
| 議事録の内容 | 🔴 必須 | 要約または全文 |
| 担当者 | 🟡 任意 | 省略時は佐藤篤也 |
| ネクストアクション | 🟡 推奨 | あれば `- [ ]` 形式で |

録画日が不明なら「録画日を教えてください（例: 2026-04-20）」と質問すること。今日の日付で仮置きしてはいけない。

## 1. 実行計画の提示（Plan mode 強制）

書込前に以下のフォーマットで実行計画を提示し、ユーザー承認を取得:

```
━━━━━━━━━━━━━━━━━━━━━━━━━
📝 Notion議事録エントリ作成 — 実行計画

対象DB: 議事録DB (2bc48452-bd25-8007-838c-c3c707c953c1)
タイトル: [候補者名] [面談種別] - [録画日]
録画日: YYYY-MM-DD  ← 必須フィールド
担当者: <name>
内容: <要約3行>

この内容で作成してよろしいですか？
  • 作成する → 「OK」
  • 修正する → 修正内容を指示
━━━━━━━━━━━━━━━━━━━━━━━━━
```

**承認後のみ Step 2 に進む。**

## 2. Notionページ作成（hr-support MCP: create_notion_child_page）

- **database_id**: `2bc48452-bd25-8007-838c-c3c707c953c1`（議事録DB固定）
- **日付プロパティ**: 「録画日」フィールドに入力する。「日付」フィールドではない
- **タイトル**: `[候補者名] [面談種別] - [録画日]` 形式

## 3. 内容の構造化

以下のフォーマットでページ本文を作成:

```
## 基本情報
- 日時: [面談日時]
- 参加者: [担当者] / [候補者]
- 種別: [IS/FS/最終/カジュアル等]

## 議事録
[内容を箇条書きで]

## ネクストアクション
- [ ] [アクション1]
- [ ] [アクション2]
```

## 4. 完了報告

```
✅ Notion議事録エントリ作成完了
  ページURL: <url>
  タイトル: <title>
  録画日: <recording_date>
  DB: 議事録DB
```

## 失敗時の対処

| エラー | 対処 |
|---|---|
| `object_not_found` (database) | database_idが誤りか権限不足。Notion側でDB共有設定を確認 |
| `validation_error` (録画日) | 録画日フィールドが別名になっている可能性。Notion側のプロパティ名を確認 |
| `rate_limited` | 5秒待機後にリトライ |
| `unauthorized` | NOTION_TOKEN失効。`.env` を確認 |

## ❌ してはいけないこと

- 録画日を空白のまま作成すること（絶対禁止）
- 録画日を「今日の日付」で自動補完すること（必ずユーザー確認）
- 「日付」フィールドに録画日を入力すること（「録画日」フィールドが正しい）
- 確認なしに即時作成すること
