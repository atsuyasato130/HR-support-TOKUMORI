企業情報をWebリサーチしてSFに補填する。対象=$ARGUMENTS

## 難易度スケール（判定基準）

| ランク | 目安 | 例 |
|---|---|---|
| **S** | 上場大手・著名グローバル。倍率50x以上・内定率<2% | エムスリー・DeNA・サイバーエージェント・リクルート |
| **A** | 中堅〜大手ベンチャー。倍率20〜50x・内定率2〜5% | ネクストビート・エグゼクション・ノースサンド・GA technologies |
| **B** | 成長スタートアップ・採用積極中堅。倍率5〜20x・内定率5〜15% | ジールコミュニケーションズ・エール・オーリーズ |
| **C** | 小規模・採用超積極企業。内定率15%以上 | 少人数スタートアップ・地方中小 |

Field2__c格納形式: `"B｜中堅ベンチャー・面接3回・書類〜最終で約1ヶ月"` （ランク + 一行理由）

---

## Step 1: 対象企業の特定

引数が「全社」または空の場合:
```
search_salesforce("SELECT Id, Name, Website, Description, Field2__c FROM Account WHERE IsPersonAccount = false AND (Website = null OR Website = '') ORDER BY LastModifiedDate DESC LIMIT 50")
```

引数が企業名の場合:
```
search_salesforce("SELECT Id, Name, Website, Description, Field2__c FROM Account WHERE IsPersonAccount = false AND Name LIKE '%<企業名>%'")
```

対象企業リストをユーザーに確認表示:
```
━━━━━━━━━━━━━━━━━━
🎯 補填対象企業
━━━━━━━━━━━━━━━━━━
1. 株式会社〇〇（Website: なし / Description: なし）
2. 株式会社△△（Website: あり / Description: なし）
...
合計 N 社を補填します。
━━━━━━━━━━━━━━━━━━
```

## Step 2: Webリサーチ（firecrawl_search）

企業ごとに以下の検索クエリで調査:

```
firecrawl_search(query="<企業名> 採用 公式サイト 求人", limit=5)
```

結果から:
1. **公式URL**: `https://` から始まる企業ドメイン（採用ページは除く。corp./company./careers./ を省いてルートドメイン）
2. **事業概要**: 何をしている会社か・特徴
3. **選考フロー**: ES→GD→面接回数など（記載ある場合）
4. **従業員規模**: 規模感（難易度判定に使用）
5. **設立年・上場有無**: 認知度・安定性判定に使用

追加検索（必要な場合）:
```
firecrawl_search(query="<企業名> 採用倍率 選考 面接 難易度", limit=3)
```

## Step 3: 難易度判定とDescription生成

収集情報をもとにClaude（自分）が判定・生成:

### 難易度判定ロジック
- 上場企業 → S または A
- 従業員500名超 → Aに傾ける
- 採用倍率が明記されている場合はそれを優先
- 「積極採用」「大量採用」ワードあり → B〜C
- 知名度が低い・設立数年以内 → C寄り

### Descriptionフォーマット（既存フォーマットを厳守）

```
🎯 一言で言うと
「<企業キャッチコピー>」

📌 事業内容
<事業概要 2〜4文。具体的な事業・数字・社会的意義を含める>

✅ 就活生に知ってほしいポイント
・<ポイント1（職種・キャリアパス等）>
・<ポイント2（企業文化・働き方等）>
・<ポイント3（数字・実績・待遇等）>

📊 選考難易度
<ランク>｜<理由一文（規模・選考回数・内定率の目安等）>
```

選考フローが判明している場合のみ追加:
```
📋 選考フロー
<フロー文字列>
```

## Step 5: SF更新実行

「はい」確認後、企業ごとに `update_salesforce_record` MCP で更新:

```
update_salesforce_record(
    object_type="Account",
    record_id="<SFのId>",
    fields_json='{
        "Website": "<URL>",
        "Field2__c": "<難易度テキスト>",
        "Description": "<生成したDescription>"
    }'
)
```

**重要ルール**:
- `Website` が既に入っている場合は上書きしない
- `Description` が既に入っている場合は `📊 選考難易度` セクションのみ末尾追記（他は変えない）
- `Field2__c` は常に上書きOK（難易度は再評価して更新して良い）
- `Field3__c` は絶対に触らない

## エラー対処

| 問題 | 対処 |
|---|---|
| firecrawl検索でURLが特定できない | `「公式URL不明（要手動確認）」`として一覧に残す |
| SF Accountが複数ヒット | 候補を提示してユーザーが選択 |
| SF更新エラー | スキップして次社へ。最後にエラー一覧表示 |
| Descriptionが長すぎる（32KB超） | 省略して上位セクションを優先 |

## ❌ してはいけないこと

- `Field3__c`（Boolean）を変更すること
- 既存の`Website`を勝手に上書きすること
- 既存Descriptionの`🏢`や`📅`セクション（日程情報）を削除すること
- ユーザー確認なしでSF更新を実行すること
