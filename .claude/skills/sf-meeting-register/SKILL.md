Salesforce面談登録フローを実行する。

## 🚨 重要: このスキルは `/sf-register` コマンドに委譲します

SF面談登録の **正式な詳細手順は `/sf-register` コマンド** に定義されており、以下を含む完全なフローを実行します：

- Google Calendar自動連携（面談日取得）
- SF + スプレッドシート並列検索
- 40+ 必須フィールドの入力ルール
- 学部系統の自動判定
- 紹介者テーブルからの自動マッピング
- 直紹介メモのフォーマット生成
- 登録後の自動チェック（`/sf-check` 相当）

## 実行指示

ユーザーの発話を受けたら、**必ず以下を実行してください**:

1. **`/sf-register` コマンドを呼び出す** — 面談記録テキストを引数として渡す
2. コマンド実行中に Plan mode で計画を提示し、ユーザー承認を得る
3. Step 0〜8 を順に実行する（/sf-register の定義通り）
4. 完了後は必ず `/sf-check` で検証する

## 事前確認が必要な情報

ユーザーが提供していない場合は、`/sf-register` 実行前に質問する:

- 面談記録テキスト（tldv等・必須）
- 面談日（不明なら Google Calendar から推定 → ユーザーに確認）
- 担当者名（省略時は佐藤篤也）

## 失敗時の対処

| エラー | 対処 |
|---|---|
| `REQUIRED_FIELD_MISSING` | 不足フィールドを補完して再実行（/sf-register Step 5 参照） |
| `INVALID_FIELD` | フィールド名のtypo確認 |
| `DUPLICATE_VALUE` | 既存学生の可能性。Name検索で既存レコードを再確認 |
| 重複候補が複数 | ユーザーに選択を促す |
| スプレッドシート取得失敗 | 「取得できませんでした」と記録して続行（/sf-register Step 2） |

## 最終チェック

登録完了後、以下を必ず確認:
- [ ] Account ID / Task ID が取得できている
- [ ] `/sf-check` の結果が全て ✅
- [ ] 大学名・学部系統・内定先企業・ガクチカレベルに NULL が残っていない

問題があれば即座に `update_salesforce_record` で修正する。
