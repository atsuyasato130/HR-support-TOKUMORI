## §1. ビジョン・テーゼ（北極星）
AIが情報収集を無料化した。**「情報を出版して注目を集める」型メディアは死ぬ**（ノウハウ・一般論・公開データの要約はAIが一瞬で出す）。だから定義を更新する。

> **新しいメディアとは「情報の出版社」ではなく「成果に責任を持つ、生きたネットワーク」である。**
> スローガン：**「読むメディア」から「受かるメディア」へ。**

情報を渡すのをやめ、**判断と成果**を渡す。「どう書くか」ではなく「あなたはどこを受けるべきか」を示し、**実際に内定させる**。

**最終目的**：学生の就活データを束ね、それで企業の採用を支援し、**世の中の企業をより良くするデータカンパニー**になる。

## §2. 6原則 ＋ 4層モデル ＋ 体験層（AIに殺せないものだけで作る）

### 6原則
1. 情報ではなく**判断と成果**（次の一手＋内定という結果に責任を持つ）。
2. **コンテンツは餌・データは頭脳・ネットワークは体・成果は証明**（記事/AIは集客と伴走＝軽く保つ）。
3. **"みんな向け"をやめ"あなた専用メディア"に**（ログイン後トップ＝「あなたの就活」）。
4. **使うほど価値が増す（フライホイール）**＝積み上げた自分とスカウト機会で、価値によるロックイン。
5. **両側に信頼で責任**（学生＝本物の成果/正直なデータ、企業＝質の高い候補/クライアント信頼）。信頼はAIが発行できない通貨。
6. **体験そのものが堀**＝生きて動き、トレンドと本人に合わせて姿を変えるUI。

### 4層モデル（どこで稼ぎ・どこが堀か）
| 層 | 役割 | 位置づけ |
|---|---|---|
| 餌＝コンテンツ/AIツール | 安く学生を集める入口（AIO/SEO）・伴走 | 模倣される。**堀ではない・軽く保つ** |
| 頭脳＝独自データのフロー | 誰がどこまで進んだ/受かった/反応したかを捕捉 | **最大の堀**（非公開・取引由来） |
| 体＝二面ネットワーク | 検証済み学生 ↔ 企業スカウト ↔ 内定 | 流動性が堀 |
| 証明＝成果 | 内定・スカウト成立・伴走実績 | 存在理由（情報でなく結果） |

### 体験層＝動き × 適応 × パーソナライズ（UIも堀）
- **未ログイン＝トレンド適応（ビルド時）×上質モーション**／**ログイン＝フル・パーソナライズ（動的）**。
- 静的テキストのAI回答には、この"生きた体験"は作れない。クローラビリティ厳守（§9・D8）。

## §3. 不公平な強み ＆ 北極星KPI
- **不公平な強み**：二面ネットワークの最難関＝企業側の流動性（鶏卵問題）。**Tokumoriは本業で人事ネットワーク（企業側）×新卒コーチング（学生側）の両方を既に持つ唯一の立場** → AIネイティブ新規参入者に作れない堀。
- **最優先KPI＝一次データ収集の最大化**：①会員登録数（特に**認証済み・実名**の質の高い会員）②データ密度（1人あたりES数/選考実績/テスト/継続日数）。
- 従KPI：内定数 / スカウト成立・返信率 / プロフィール充実度 / 継続率 / AI引用率（入口効率）。
- **設計判断軸**：新機能は必ず「頭脳/体/証明のどれを太らせるか」「登録が増えるか/データが濃くなるか/継続するか」で評価。餌しか太らせないなら優先度を下げる。判断に迷ったら：**「このサービスが消えたら学生と企業は本当に困るか？困る理由が"情報"なら死ぬ、"つながり・成果"なら残る。」**

## §5. 中核＝トクモリ偏差値（TokumoriScore）＝全データの集約点
SF準拠の**7軸**を学生に自動割当→合成→**会員母集団に対する偏差値**（平均50・SD10）。

| 軸 | 主な入力源 |
|----|-----------|
| 学歴 | 大学・学部 |
| 実行力 | 選考到達レベル(ABABA)・STARのResult・**長期インターン成果(受入先入力エビデンス)** |
| 対人力 | AI面接・性格テスト・GD実績 |
| 人格 | 性格テスト |
| 思考力 | Webテスト・AI面接の論理性 |
| 就活習熟度 | プロフィール完成度・ES数・自己分析・スケジュール運用・模試回数 |
| ガクチカレベル | 自己分析STARの質・ES内容・AI添削スコア |

- **更新**：月1回のAI面接＋データ変化で随時。履歴を保持し「偏差値の伸び」を可視化（学生の成長実感＋掲載企業の母集団の質の証明）。
- **会員内偏差値**：プロト＝決定論式＋ダミー母集団／本番＝Supabaseで真の母集団集計（夜間バッチ）。
- **用途**：自己診断／**偏差値ゲートのスカウト・特別オファー**／**模試型 合格可能性判定**／不足者の長期インターン誘導／掲載企業ダッシュボード。
- **コンプラ**：差別的選別の禁止。学歴等センシティブ軸を含むため、企業ゲートは「準備度の目安」と明記し最終判断は企業。断定/保証しない（YMYL）。

## §6. プロダクト全体像（3面）＋ 公開/会員の線引き（AIO堀）
- **学生**：情報収集 → プロフィール蓄積 → 管理ツール利用 → 偏差値 → スカウト受信 → AI対策 → エントリー まで完結。
- **企業**：0円掲載 → 企業ページ自社編集 → エントリー受付 → 選考進捗×偏差値でスカウト/特別オファー → 掲載企業ダッシュボード。
- **エージェント**：同意済みプロフィールへスカウト・支援。

| 公開（AIに引用される＝集客） | 会員裏（登録誘導・収益・noindex） |
|---|---|
| 記事の要点/結論/一次データ要約・公式統計・企業の基本情報/公開統計・人事のリアルの**ティーザー** | 記事フル・ESフル・AIツール・スカウト・マイページ・偏差値・人事のリアル本文・各特典 |

> AIに引用されたいものは公開、囲い込み・収益化は会員裏（会員裏はAIに引用されない＝堀が効く）。**会員データは全て noindex・SSR非露出**（実装済の InsiderArticle / MemberGate 方式）。

## §8. AIOメディアの中核（餌＝集客の入口）
### 記事の型システム（全記事＝AI抽出ユニット・上から順）
H1（質問形式）→ ConclusionBlock（冒頭3-5行・確認済み事実のみ）→ DataPoint（数値/サンプル数/時期）→ 本文（見出し直下に答え→根拠・2-3文）→ FaqBlock（実在時のみ・表示とschema一致）→ SourceCitation。
- 記事タイプ5種：`how-to`／`survey`（DataPoint必須・堀）／`comparison`（**AnswerMatch必須＝答え合わせ＝記憶に残る一点**）／`glossary`／`interview`。
- 1記事＝1検索意図＝1質問。薄い記事を多数で階層を太らせる。

### 一次データ・パイプライン（最大の堀）
2系統（人事側／就活生側）＋**答え合わせ**（学生が良いと思うES vs 人事が評価するES のギャップを `AnswerMatch` で数値化）。収集SOP：設問設計→取得（本業ネットワーク＋サイト内SurveyWidget）→**同意取得/記録**→匿名化/統計化→人間レビューゲート→構造化→記事化。**個別企業のホンネは生公開しない**（業界傾向に集約）。

### AIクローラ許可（引用されたい＝全許可・D5）
robots.txt は学習/引用とも全許可＋2026新UA（**OAI-SearchBot / Claude-SearchBot / Perplexity-User**）。引用させたい記事はログイン/ペイウォール/JS依存の裏に置かない。CloudflareのAIボット既定ブロックに注意。詳細UA＝`就活メディア_設計仕様書_v1.md` §7。

### JSON-LD（土台・過信しない）
`lib/schema.ts` で**表示を単一の真実として自動生成**（drift防止）。Organization/WebSite/Article/FAQPage/BreadcrumbList/Dataset/Person。Microdata/RDFa不使用。

### エンティティ・信頼シグナル
著者の権威性（人事経験等）＋運営者明確化＋エンティティ階層URL＋**外部言及（本業人脈・立ち上げ速度の律速）**。

## §11. 技術アーキテクチャ
- **フロント**：Next.js16（App Router・Turbopack）/ React19 / TypeScript / Tailwind v4（@theme CSS-first）。SSG基本＋ISR。フォントはビルド時取得をやめ`<link>`読込（オフラインでもビルド成立）。`params/searchParams`はPromise（await）。会員データは `robots:{index:false}` ＋ SSR非露出。
- **永続化（プロト→本番）**：localStorage薄層（`profile-store`/`member-store`/`scout-store`＋新設 es/schedule/bookmark/score/assessment/obog/company-page/points 等）→ **Supabase** に差し替え（呼び出し側はストア関数のみ依存）。
- **バックエンド（後フェーズ）**：Supabase（Postgres+Auth+Storage+**RLSで同意ベース公開を強制**）／Stripe（掲載0円/スカウト/特別オファー/成功報酬/データプロダクト）／Anthropic API（添削/AI面接/マッチング・参考値・人間検証）／LINE Messaging API（通知）／Resend（メール）／Vercel Cron（学習ループ）。
- **CMS**：microCMS（`lib/cms.ts`インターフェース化・env無ければダミー自動フォールバック）。
- シークレットは `.env.local`（同期外・コミット禁止）。`.stignore`済。

## §12. デザインシステム＆原則（リッチ・脱AI・毎日開きたい）
- **ブランド**：主色 Tokumoriレッド `#AF322C` ＋黒。背景＝温かいオフホワイト `#faf7f1`。**アクセントは赤1色**。ember `#e2574a`。
- **フォント**：見出し＝**Shippori Mincho（明朝ディスプレイ）**＋本文＝Zen Kaku Gothic New＋数値＝IBM Plex Mono。**禁止**＝Inter/Roboto/Arial/system-ui/Space Grotesk・紫グラデ・均等パステル・絵文字依存UI。
- **方針（強い再フィードバック）**：現状は「チープ」。**リッチで就活生のテンションが上がり、毎日開きたくなる**質感へ（継続＝データ収集に直結）。編集×データの高級感（明朝＋上質実写＋ヘアライン＋特大の実数タイポ＝雑誌/ハイブランド誌面）。AIっぽい平均値・ガラス/グロー/角丸ダッシュボードは禁止。
- **モーション**：ロード時に主役の動き1つ（staggered reveal）。`prefers-reduced-motion`尊重。**モバイルファースト**。
- **記憶に残る一点**：答え合わせ(AnswerMatch)＋特大の内定率タイポ＋偏差値の伸び。
- 品質チェック：①平均値でないか ②禁止フォント不使用か ③アクセント2色以上でないか ④記憶に残る一点があるか ⑤"毎日開きたい理由"が画面にあるか。詳細＝[[feedback_richer_media_design]]／CLAUDE.mdフロントエンドUIルール／`/frontend-design`。

## §13. リアル数字でブラックボックスを開ける
就活の不透明（みんなどれだけ動いてる？内定持ってる？募集は締まった？）を**集めた実データで可視化**＝健全な焦り→行動。
- 2層：①公式統計（就職みらい研究所等・今＝`lib/stats`）②**自社会員データ（会員N≥10,000で開放）**＝同卒年の動き出し率/内定保有率/平均ES数/**募集締切済み企業数・%**/偏差値分布の中の自分。
- 長期インターン側：**経験者は世代の◯%**（希少性＝価値）／AI代替されうる仕事の割合等で「今動く意味」を提示。
- 原則：すべて実データ由来・出典/時点明記。捏造ライブ禁止（YMYL）。N不足時は公式統計のみ＋「会員データはまもなく解放」。

## §14. 信頼・継続・本人確認・ポイント・飛び対策・信用値の外部展開
- **本人確認(M23)**：大学メール認証/学生証/実名（ダミーネーム不可）→「認証済み」バッジ→評価/特典/スカウト到達が上がる。登録期間も評価。データ品質とLTVの根幹。
- **ポイント(M24)/プレミアム招待(M25)**：活動でポイント→ギフト券/提携交換。上位は限定招待（育つと招かれる＝継続動機）。
- **飛び対策(M26)**：無断キャンセルを企業が記録→信頼スコア低下→評価が上がらない（誠実な学生が報われる）。本人通知・異議申立必須。
- **学生信用値(個信)の外部展開(M27・最大拡張)**：トクモリ偏差値＋信頼スコア＋実績を「信用」として、**本人同意のもと**就活外（不動産＝良い物件/保証会社不要・金融・賃貸保証・優待）へ。Timee型の発想転換。**個情法/信用情報規制/割賦販売法・貸金業法等＝要弁護士。設計と同意基盤は早期に仕込み、実装は最終フェーズ。**

## §15. マネタイズ（データが溜まるほど商品が増える）
掲載＝**基本0円**（母集団形成）。課金：スカウト／**特別オファー枠**（偏差値上位への限定）／成功報酬／**データプロダクト**（母集団分析・採用ベンチマーク：自社採用学生の偏差値推移 vs 業界平均）／長期インターン送客／将来：信用値外部提携のレベニューシェア・採用最適化コンサル。

## §16. コンプライアンス（機能ではなく前提条件）
YMYL＋人材ビジネス。①学生データは本人入力＋**公開範囲トグル**（同意分のみ企業に・RLS強制）②受けた学生の内定率/落ちた傾向は**完全匿名・統計化のみ**（個人特定不可）③人事の声は企業許諾④AI判定は**参考値・断定しない**・人間検証⑤掲載/スカウト課金は**職業安定法・個人情報保護法・景表法を着手前に法務確認**⑥信用値の外部提供は信用情報規制＋本人同意＋説明可能性＋差別禁止⑦データ提供同意（ES等の二次利用）は利用規約＋同意ログ。**実運用前に弁護士確認必須**。
