## 2. Slack Bot Token 設定（5分）

### 200_設定 タブ
B20 セル「Slack Bot Token」に `xoxb-...` で始まるトークンを入力。

### Bot Token の取得方法
1. https://api.slack.com/apps を `record@tokumori.co.jp` で開く
2. 既存の Bot App を選択（ユーザーが招待したアシスタント）
3. 左メニュー **OAuth & Permissions** → `Bot User OAuth Token` をコピー
4. このトークンを 200_設定 B20 に貼り付け

### 必要な Bot Scopes
- `chat:write` (メッセージ投稿)
- `channels:read` (チャンネル確認)

C0A59L3LU12 チャンネルに Bot を招待済みであることをご確認ください。

## 7. raw_data 新フェーズ対応について

raw_data の以下の新規列は **detectPhaseDefs が自動で拾います**:
- 説明選考会【不合格】
- 一次面接【辞退】
- 1day後面談（人事/現場）
- 2day後面談（人事/現場）
- 二次面接【参加】/【不参加】/【合格】/【不合格】
- 最終選考【参加】/【不参加】/【合格】

グループ分類も自動:
- 【不合格】【不参加】【辞退】 → `exit`（離脱集計＋累積から除外）
- 【合格】内定 内定承諾 → `pass`
- 【予約】→ `reserved`
- 後面談 → `interview`（ファネルから除外）
- それ以外 → `main`

すべて既存の `detectPhaseDefs` ロジックでカバーされています。

## 10. 既知の制約

- Slack Bot Token は手動設定が必要（OAuth トークンは GAS から取得不可）
- 月次レポートV2は前月分集計のため、月初トリガーは前月のデータを使う
- QuickChart.io への依存（無料・無制限・APIキー不要）
- Gemini API キーは Script Properties の `GEMINI_API_KEY` に設定済の前提
