# LG/CA 面談連携システム 運用マニュアル

最終更新: 2026-05-25 / バージョン: v5.7

---

## 1. 概要

学生面談の登録〜確定〜実施〜キャンセル/飛び/重複報告までを **Slack ＋ Google Spreadsheet ＋ Google カレンダー** で一元管理するシステムです。

- 連携シート: https://docs.google.com/spreadsheets/d/1NmJXivFLnPC-Nb50ANZx2gZWT4vPneOm7XVF-B4t698/edit
- 本番チャンネル: `#test-lg-ca-連携-new` (C0ATUR7P3MK)
- 飛び報告チャンネル: C0A4VGTAPRB
- GAS プロジェクト所有者: `record@tokumori.co.jp`

---

## 2. 役割

| ロール | 担当 |
|-------|-----|
| **LG** (Lead Generator) | 学生獲得・面談登録・日程入手 |
| **CA** (Career Adviser) | 面談実施・確定処理・確認応答 |

---

## 3. 全体フロー

```
[1] LG 学生から日程取得
     ↓
[2] Slack ショートカット「面談登録」を起動
     ↓
[3] モーダルで情報入力（学生名・卒年・SNS種別・候補日程・つなぎ方等）
     ↓
[4] 自動で CA カレンダー照合
     ↓ ┌──────────────────────────┐
     ├─ 空きあり → CA 自動アサイン・Meet URL 生成 → 「実施予定」フェーズ
     └─ 空きなし → 「日程調整中」フェーズ + @CA グループメンション
     ↓
[5] 実施または各種報告（CA確認・リスケ・キャンセル・飛び・重複）
```

---

## 4. 面談登録手順（LG操作）

### Step 1: ショートカット起動

Slack 入力欄左下の ⚡ メニュー → **「面談登録」** をクリック
（または本番チャンネル `#test-lg-ca-連携-new` の「+」ボタンからショートカット呼び出し）

### Step 2: モーダル入力

| 項目 | 入力 |
|------|-----|
| 予約モード | `通常モード` / `自分のカレンダーで直接予約（兼任）` |
| LGメンバー | 自動セット（変更可） |
| 学生アカウント名 | `tanaka_taro` 等 |
| 卒年 | 27卒〜30卒 |
| **SNS種別** | Instagram / X / TikTok / YouTube / Threads / リファラル / その他 |
| **└ その他選択時の記述** | その他を選んだ場合の自由記述 |
| **つなぎ方** | 友達繋ぎ / 先輩繋ぎ / インターンの人繋ぎ / その他（自由記述） |
| **└ その他選択時の記述** | 同上 |
| 志望業界 | 業界選択 / その他 + 自由記述 |
| 相談内容 | 自己分析等 / その他 + 自由記述 |
| 候補日①②③ | 日付 + 時間 |

#### 候補時間のフォーマット（自由記述OK）

| 入力例 | 解釈 |
|--------|------|
| `10:00~12:00` | 10:00〜12:00 |
| `10-15` | 10:00〜15:00 |
| `10時以降` | 10:00〜23:00 |
| `午前中` | 9:00〜12:00 |
| `午後` | 13:00〜23:00 |
| `夕方` | 17:00〜23:00 |
| `終日` | 8:00〜23:00 |

### Step 3: 送信

→ シートに行追加 + Slack 本番チャンネルにスレッド投稿

## 5. CAアサインルール

### 営業可能時間（ビジネスアワー）

| CA種別 | 平日 | 土日 |
|-------|------|------|
| `@tokumori.co.jp` ドメイン (社員) | **9:00 - 20:00** | NG |
| その他（業務委託/個人Gmail等） | **9:00 - 23:15** | 9:00 - 23:15 |

### マッチング条件

- 候補時間内で `45分` の空き枠を `15分刻み` で探索
- カレンダーに **何かしらの予定があれば一切ブッキング不可**（完全空きのみ）
- アクティブCAリスト順に検索 → 最初に空きが見つかったCAでアサイン
- CA 自動アサイン失敗 → **「日程調整中」フェーズ** で待機

## 6. スレッド内ボタン

ボタン配置: **左 LG用 / 中 共通 / 右 CA用**

### 実施予定フェーズ

```
[🔁 リスケ依頼] [❌ キャンセル] [🚫 飛び報告]  [⚠️ 重複報告]  [✅ CA確認]
       LG 用                              共通          CA 用
```

| ボタン | 操作者 | 動作 |
|--------|--------|------|
| **🔁 リスケ依頼** | LG | モーダルで新候補入力 → **旧CAカレンダー予定削除** → CA 再マッチング |
| **❌ キャンセル** | LG | フェーズ「キャンセル」化 + **CA カレンダー予定削除** + スレッド通知 |
| **🚫 飛び報告** | LG | フェーズ「飛び」化 + 飛びチャンネル通知 + カレンダーに「飛び」注記 |
| **⚠️ 重複報告** | LG/CA | モーダルで詳細入力 → フェーズ「重複（要整理）」化 |
| **✅ CA確認** | CA | フェーズ確認済み + CA本人にDM通知 |

### 日程調整中フェーズ

```
[🔁 日程再回収] [❌ キャンセル] [🚫 飛び報告]  [⚠️ 重複報告]  [📅 日程入力]
       LG 用                              共通          CA 用
```

| ボタン | 操作者 | 動作 |
|--------|--------|------|
| **🔁 日程再回収** | LG | モーダルで新候補入力 → CA 再マッチング |
| **❌ キャンセル** | LG | フェーズ「キャンセル」化 |
| **🚫 飛び報告** | LG | フェーズ「飛び」化 |
| **⚠️ 重複報告** | LG/CA | 重複処理 |
| **📅 日程入力** | CA | モーダルで確定日時手入力 → フェーズ「実施予定」化 + カレンダー登録 |

## 7. リマインドフロー

「日程調整中」フェーズで登録から:

| 経過時間 | 動作 |
|---------|------|
| **12時間** | @CA グループメンション + 希望日程付きリマインド(1次) |
| **24時間** | @CA グループメンション + 「優先対応」リマインド(2次) |
| **36時間** | 自動「調整不可」化 + 最終通知 |

---

## 8. フェーズ一覧

| フェーズ | 意味 | 遷移元 |
|---------|------|--------|
| **日程調整中** | CA未確定の初期/再調整状態 | 初回登録 or 日程再回収 or リスケ |
| **実施予定** | CA確定済み | CA自動マッチング成功 or 日程入力 |
| **実施済** | 面談完了 | SF同期で自動 |
| **キャンセル** | 学生事前キャンセル | LGがキャンセルボタン押下 |
| **飛び** | 学生当日不参加 | LGが飛び報告ボタン押下 |
| **調整不可** | 36h経過で日程確定できず | リマインド自動遷移 or 手動 |
| **重複（要整理）** | 重複検知 | 重複報告 or 事前重複チェック |
| **リスケ** | 日程変更要求 | (履歴管理用、現在は日程調整中に統合) |

## 12. メンバー管理 (CA / LG)

v5.7+ で **1枚の「メンバー」シート** に統合。**左ブロック(A〜G列) = CA / 区切り列(H) / 右ブロック(I〜N列) = LG** の左右分割レイアウト。手動追加は **Slack ショートカット「メンバー追加」** から1分で完了します。

### シート構成（1シート / 左右分割）

| ブロック | 列 | 内容 |
|---------|----|------|
| **左(CA)** | A | ID (CA001〜・自動採番) |
| | B | 名前 |
| | C | メール (カレンダーID 兼用) |
| | D | Slack User ID |
| | E | 区分 (社員/業務委託/個人Gmail/27卒/28卒/29卒/30卒/メンター/その他) |
| | F | ステータス (アクティブ / 共有待ち / 非アクティブ) |
| | G | 備考 |
| 区切 | H | (空白列・視覚的セパレータ) |
| **右(LG)** | I | ID (LG001〜・自動採番) |
| | J | 名前 |
| | K | Slack User ID |
| | L | 区分 |
| | M | ステータス |
| | N | 備考 |

### メンバー追加手順 (Slackから)
1. Slack 入力欄左下「+」または ⚡から **「メンバー追加」** を選択
2. モーダルで入力:
   - **役割**: LG / CA / 兼任
   - **Slackユーザー**: workspace からドロップダウン検索
   - **区分**: 社員 / 業務委託 / 個人Gmail / 27卒 / 28卒 / 29卒 / 30卒 / メンター / その他
   - **メール** (任意): 空欄なら Slack profile から自動取得
   - **ステータス**: アクティブ / 共有待ち / 非アクティブ
   - **備考** (任意)
3. 送信 → 該当ブロックの末尾に自動追記
   - ID は自動採番 (CA014 / LG039 等)
   - 名前・UID は Slack から自動解決
   - **兼任** の場合は 左右両ブロックに同じUIDで自動登録
4. 登録者に DM で「登録完了」通知

### CAアクティブ化フロー
1. メンバー追加 (ステータス=共有待ち) でSlack登録
2. CA本人に `record@tokumori.co.jp` のカレンダー共有を依頼 (権限「予定の変更」)
3. 共有完了確認後、メンバーシートのステータス列(F)を「アクティブ」に手動変更
4. 30分以内に空き枠カレンダーへ自動反映 (手動も可)

### 注意事項
- 卒業/退職時は **行削除せずステータスを「非アクティブ」** に変更 (過去履歴の参照保持)
- 兼任CA/LGは左右両ブロックに同UIDで存在することを許容 (片方だけ非アクティブ化も可)
- 左ブロック(CA) と右ブロック(LG) は **独立して伸びる** — CA行が13行・LG行が38行という不均衡もOK

## 13. 重複検知

事前重複チェック: 同じ `学生アカウント名 + SNS種別` の既存登録があれば、シートに行追加せず LG にエフェメラル通知。

```
⚠️ 面談登録は成立しませんでした（重複のため）
学生: tanaka_taro（Instagram経由）
既存登録: #0042（登録日: 2026-05-22 / シート: 連携メイン_27卒）
```

## 14. 運用上の注意

- カレンダーの実体に依存するため、CAは「予定の変更」権限で共有が必須
- 「ブロック」「定例」など全ての予定がブロッカー扱い（例外なし）
- リスケ・キャンセル時のカレンダー予定削除は **summary `#NNNN` で検索** して実行
- Slack 3秒応答制限: モーダル開閉が遅い場合は再度ボタン押下

## 15. トラブルシュート

| 症状 | 確認ポイント |
|------|------------|
| ショートカット押下しても何も起きない | Slack App の Request URL に `?token=...` が付いているか確認 |
| モーダルが開かない | GAS実行ログで `views.open` エラー確認 |
| カレンダー登録失敗 | 該当CAが record@ に編集権限共有しているか |
| 日程調整中から進まない | 候補時間が CA ビジネスアワー外でないか |
| シートのNo.が4桁でない | GASエディタで `applyRowNoFormatAllSheets` 実行 |
