`DECK_GUIDE.md`（作り方・規律）/ `ANTI_AI_TELLS.md`（禁止則）と併用。エンジン＝`build_pitch_deck.py`（MODE×SPEC）・研修のみ `build_slides.py`。

## 1. MECE分類（聴衆 × 目的 → MODE）

### A. 社外向け（お金・意思決定が動く順）
| # | パターン | 聴衆 | 目的 | MODE | 備考 |
|---|---|---|---|---|---|
| A1 | 営業・提案書 | 顧客企業の決裁者 | 受注 | **edit×BRAND=red**（明るい提案）or **consult**（紺・重厚） | 顧客がコンサル文化なら consult、事業会社なら edit |
| A2 | 会社紹介・ファクトブック | 見込み客/パートナー/メディア | 信頼形成 | **edit×red**（メルカリFact Book型） | 巨大数字＋出典・英日章扉・目次 |
| A3 | 採用ピッチ・カルチャーデック | 転職/就活候補者 | 応募・承諾 | **block×red** or edit | SmartHR/LayerX型。写真・カルチャー厚め |
| A4 | IR・実績/経営報告（社外） | 株主・銀行・取引先 | 説明責任 | **consult** | 出典・免責厳格。ウォーターフォール/実績表 |
| A5 | 学生向け説明会 | 学生 | 興味喚起 | **soft**（明朝・温かい） | とくもり就活と同世界観 |
| A6 | セミナー・登壇 | 業界公衆 | 認知/ブランド | **edit** or **aura** | 1スライド1メッセージ徹底・特大タイポ |

### B. 社内向け（近い順）
| # | パターン | 聴衆 | 目的 | MODE | 備考 |
|---|---|---|---|---|---|
| B1 | 経営会議・役員報告 | 経営陣 | 意思決定 | **cool** or consult | データ前面・結論先出し・1枚サマリー |
| B2 | 全社共有・キックオフ | 全社員 | 士気/方針浸透 | **vivid** or block | 大胆・記憶に残る一点 |
| B3 | 研修・オンボーディング | 新人/若手 | 習得 | **build_slides.py THEME=training_v2** | 41モジュール既存。演習・図解 |
| B4 | 定例レポート・振り返り | チーム | 進捗共有 | **light** | 手早く・表とチャート中心 |

**判定フロー**: ①社外か社内か → ②目的（売る/伝える/採る/報告する/教える）→ 表のMODE。迷ったら A1=edit×red / B1=cool。

## 2. 高viewデッキから抽出した設計原則（リサーチ由来・全パターン共通）

1. **巨大数字＋小さい単位・修飾語**（SmartHR/メルカリ）: キー指標は bignum で1スライド1つ。「ARR200億円」「MAU 2,384万人」式。**必ず出典/時点注記**を小さく添える。
2. **英日併記の章扉**（SmartHR型）: `Company Profile ／ 会社概要`。A2/A3で標準。
3. **目次（Index）**: 20枚超のデッキは冒頭にページ番号付き目次（メルカリ90枚型）。
4. **統一フッター**: `© TOKUMORI, Inc.` ＋ `頁 | 総頁`。社外デッキの信頼感。
5. **色は3色まで**: メイン(ブランド)＋テキスト用ソフト黒＋グレー。**純黒でなく #1A1714 系**（SmartHR #23221f 方式）。ブランド色は白地の本文テキストに使わない（面とアクセントに使う）。透明度30/60/90%で同系展開。
6. **impact × compact**: 文字を図解・グラフに置換して一目で掴めるように（文字の壁禁止＝ANTI_AI_TELLS §1）。
7. **和文フォント**: Noto Sans JP（7ウェイト・全環境）を軸に、W4基本/W7太字のウェイト設計。数字はLato等の欧文でも可。
8. **名前付きレイアウトのレパートリー**（k1LoW deck方式）: 内容とデザインを分離し、SPEC(JSON)のtypeでレイアウトを切替（本エンジンの方式そのもの）。

## 3. レイアウト・レパートリー（SPECのtype一覧＝現在実装済み）

| type | 用途 | 主なパターン |
|---|---|---|
| cover / cta | 表紙・締め | 全deck |
| section | 章扉（番号+タイトル+サブ） | 全deck。英日併記はtitleに併記 |
| bignum | 巨大数字1つ＋ラベル＋出典 | A2/A1のKPI・実績 |
| statement | 特大の主張1文 | 章間のメッセージ |
| stats | 数値カード3-4枚 | サマリー |
| points | 番号付き要点2-4 | 打ち手・機能 |
| cards | カード+タグ+説明 | サービス・事例 |
| bullets | 素の箇条書き | 最小限に |
| split | 左ロジック+右エビデンス/図（text×textは自動でHTML 2カラム図） | 高密度説明 |
| fig: bar/funnel/pyramid/before_after/formula/process/gridmatrix/timeline/ladder/cycle/waterfall | データ・構造図解（HTMLハイブリッド・出典脚注） | データページ |
| flow | ステップカード | 導入・進め方 |

**将来拡張（リサーチで有効性確認済み・未実装）**: 目次(index)型 / チーム・メンバー紹介グリッド / 写真ヒーロー / 英日章扉の自動整形 / `頁|総頁`フッター。

## 4. 使い方（slide-genスキルから）
1. 依頼を §1 で分類 → MODE決定（ユーザー指定が最優先）。
2. SPEC(JSON)を §3 のtypeで組む（1スライド1メッセージ・図解ファースト）。
3. `MODE=<mode> BRAND=<red|blue> LOGO=assets/logo_rpo_blue.png SPEC=<spec>.json python3 build_pitch_deck.py`
4. 生成後は必ずサムネ全枚目視（DECK_GUIDE §7）＋ ANTI_AI_TELLS §4 チェック。
