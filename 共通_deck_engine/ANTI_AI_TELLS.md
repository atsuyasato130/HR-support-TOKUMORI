# ANTI_AI_TELLS.md — 「AIっぽさ」を消すための禁止則・正本

## 0. 核心テーゼ（なぜAIっぽいか）
**AIは「安全・均等・バランス」を最適化する。プロの資料は「決め打ち・階層・密度」で作られる。**
AIっぽさ＝*このスライドで何が一番大事かを決めていないことの視覚的な現れ*。各スライドに編集上の主役がいないと、全部が等しく整い・等しい余白・等しい温度の文章になり「機械が並べた」感が出る。
最大のtell（llemental/Stanford HCI 2025引用）＝**「全箇条書きが同じ長さ」**。視聴者の71%はAI資料を3枚以内に見抜く。

## 1. 禁止リスト（DO NOT — これがAIの指紋）
- [ ] **紫/青のグラデ背景・発光オーラ**（"purple gradient" はAIの代名詞）
- [ ] **角丸カード＋ドロップシャドウ＋ピル**の多用（SaaS LPテンプレの匂い）
- [ ] **英語eyebrow/タグ**（IMPACT / VALUE / AWARDED / COMPANY）→ 日本語の章番号に
- [ ] **均等3カードの反復**（2/4/5は内容駆動で可。「3つ」を機械的に選ばない）
- [ ] **均一マージン・完璧な左右対称**（＝編集判断の不在）
- [ ] **全カードを同じ長さに整形**（最大のtell。1語の所と一節の所を作る）
- [ ] **幾何学サンセリフ**（Inter / IBM Plex / Space Grotesk / Roboto）＝"blanding"＝テック/AIの記号
- [ ] **中間調パレット・均等分散の多色**（主役の色を作らない）
- [ ] タイトルが**名詞ラベル**（「会社概要」「サービス範囲」）
- [ ] チャートが**ラベルなし・出典なし・注釈なし**（"分析の汗"が見えない）

## 2. 必須リスト（DO — プロの型）
- [ ] **アクションタイトル**：各スライド＝結論の完結1文。≤15語・≤2行・能動態・"プロセスでなく結論"。タイトルだけ読めば論旨が通る（horizontal logic）。例 ✗「市場規模分析」→ ✓「対象市場は2.4兆円、年18%で拡大」
- [ ] **3層ピラミッド（論理）**：タイトル＝結論／中段＝論拠／下段＝データ。※固定ピクセル配置ではない（俗説は反証済）
- [ ] **色＝ダーク中立1＋アクセント1＋グレー段階**（3〜4色まで）。アクセントは*装飾でなく強調*にのみ。残りは全てグレー3段で de-emphasis。色は注意誘導の道具。
- [ ] **フォント≤2・同一要素は全頁同サイズ**。日本語は **Noto Sans JP（無主張・中立）/ BIZ UDPGothic（UD・institutional）**。游ゴシック/MeiryoはGoogleスライド不可。
- [ ] **注釈付きチャート（think-cell流）**：データラベルをセグメント直貼り／差分・CAGR矢印／基準線／ウォーターフォール（正＝上・負＝下、小計は基準線まで）／マリメッコ（面積∝絶対値）。各図に**出典脚注**。
- [ ] **制御された密度**：本質は密でよい（"dense but never cluttered"）。削るのは非本質。補足は付録へ。
- [ ] **示唆（So-What）**：各スライドに「だから何か」を一言。
- [ ] **出典脚注を全スライド下部**に（静かなグレー小）＋ページ番号。

## 3. 棄却された俗説（ハードコード禁止＝"規則"ではない）
- 60/40の2カラム固定 … 反証(0-3)。内容駆動で可変。
- 上中下の固定ピクセル配置 … 反証(0-3)。論理階層であってレイアウト固定ではない。
- charcoal #333 / #F8F8F8 / アクセント10-20% … 具体値は未確証(1-2)。色値は文脈で。

## 4. 自己チェック（実装後・サムネ目視で問う）
1. このスライドの**主役を1つ**に絞れているか？（複数主役＝AI的均等）
2. タイトルだけ読んで**論旨が通る**か？
3. グラデ/角丸/ピル/英語eyebrow/均等3カードが**残っていないか**？
4. フォントは**institutional**（BIZ UD/Noto）か？ 幾何学サンセリフが紛れていないか？
5. 図に**データラベル・出典**があるか？ アクセントは**1図1〜2箇所**か？
6. 文章は**長さがバラついている**か？（均一＝tell）

## 5. ユーザーの言語化（追記欄 — ここに一行ずつ蓄積し、次回生成から反映）
<!-- 例: 「余白が広すぎて間延びして見える」「数字がもっと主役であってほしい」「色は紺一色で良い」 -->
- 2026-07-01「示唆（So-What）バンドの様式そのものがAIっぽい」→ ラベル付き定型バンドは廃し、結論はタイトル/本文に自然に溶かす。
- 2026-07-01「数字のフォント・大きさ・図の作り方にAIっぽさが残る」→ 等幅/テンプレ然とした数値表示・図の量産感をやめ、エディトリアルに個別設計する。
- 2026-07-01「コンサル(マッキンゼー/think-cell)型の高密度より、モダンで美しい“自社っぽい”デザインが欲しい」→ 目標は SpeakerDeck/Canva 的な現代エディトリアル。わかりやすく・見やすく・美しく・現代的。
- 2026-07-01「何を伝えたいか分かりづらいスライドがある」→ 1スライド1メッセージを更に徹底し、主役ビジュアルを大きく。
- 2026-07-02「全体的にまだAI感が抜けない。サンプル提示→FB反復で詰める」→ 一発生成でなくサンプル→FBループを標準に。
- 2026-07-02「器（茶碗）モチーフはかわいすぎる。表紙はもっとスマートでかっこよく」→ 器のベタ半円は廃止候補。細線アーク等の抽象化はOK。ブランド色は維持。
- 2026-07-02「スライド全体/背景をHTMLで作ってよい」→ ハイブリッド（HTML背景×ネイティブ文字）解禁。LayerX/SmartHR級が基準。
- 2026-07-02「図解カタログ50は良い。もっとパターンを増やして」→ 図のトンマナ（温白×墨×赤1色・palt・Lato数字・細部の精密さ）は合格ライン。拡張継続。
- 2026-07-02「背景カタログ14案（アーク/方眼/斜めカット/ドット等）は全てイケてない」→ 装飾で作る背景はNG方向。過剰な図形装飾自体がダサさ/AIっぽさになる可能性。
- 2026-07-03【確定】表紙・背景は「極限シンプル」（実物LayerX/メルカリ流）：装飾ゼロ・白・ロゴ・タイトルだけ。タイポグラフィ（サイズ/字間/palt）と余白の精度で見せる。アーク等のブランドモチーフも表紙には置かない。
- 2026-07-03【承認】図解カタログ80パターン（F01-F80）＝「完璧」。このトンマナ（温白×墨×赤1色・palt・Lato数字・罫線最小）が図の正式基準。表紙は「物足りなさはあるが一旦シンプルでOK」＝暫定確定（将来ブラッシュアップ余地）。
- 2026-07-03【確定】本番アーキテクチャ＝ハイブリッド：デザイン層はHTML品質（図・グラフ=HTML→PNG）、テキストは必ずネイティブ（編集可能であることが必須要件）。背景は白（極限シンプル）なので背景画像は不要＝文字は全部その場で編集できる。エンジン実装は `MODE=simple`。
- 2026-07-03【要件】スプレッドシート由来の数字は紐付ける：①グラフ=createSheetsChartのLINKEDで挿入（シート更新→「更新」ボタン）②表・数値=SPECに直書きせず sheet_id+range から生成時に読込（更新は再生成1コマンド）。数値の手転記を禁止。
