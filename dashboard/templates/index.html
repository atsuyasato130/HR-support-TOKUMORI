🔏 PIIポリシー

PII定義: 氏名・連絡先・面談録画・評価・財務情報・採用候補者情報を個人識別可能情報（PII）として分類する。

取扱原則: PII を含む処理には最小権限の原則を適用。外部サービスへの書込みは必要最小限とし、Human-in-the-Loop が必要な処理には承認ステップを設ける。

ログ保持: 実行ログに PII をそのまま記録しない。要約・匿名化した形式で記録する。

⚡ エージェント自律度レベル
Lv 1
対話型
動作: 会話中のみ・毎回手動で起動が必要
代表例: HRsupport 11体稼働中・全42体設計中含む
前提: Claude Code / ブラウザから指示を与える
限界: PC を閉じると停止。夜間・休日は無人運転不可
Lv 2
自律型
動作: 定期実行・スケジューラーで 24h 稼働
代表例: exec_strategy（週次レポート想定）
前提: PC 常時 ON または GAS/cron との組み合わせ
限界: PC 電源 OFF で停止。VPS 不要だが常時 ON が必要
Lv 3
常駐型
動作: VPS 常駐・Webhook 即時反応・PC 電源不要
代表例: hr_orchestrator_main（開発中）
前提: VPS + 常駐プロセス管理（pm2 等）
限界: インフラコスト・セキュリティ設計が必要

👥 チームでの利用ルール

🔐 APIキー管理: 各自の .env ファイルで管理。Slack・チャットへの貼り付け厳禁。

📋 実行ログ記録: エージェント実行後は Intel タブへログを記録する（削減時間・BU・概要）。

🔄 ステータス更新: エージェントの開発状況が変わったら Agents タブでステータスを更新する。

📝 変更履歴: コード・設定変更は必ず Activity タブ または dashboard_logger.py で記録する。

🛡️ セキュリティ: 新エージェントを追加したら Security タブのセキュリティレベルを設定する。

⚠️ private/ 分離: private/ 配下の活動・ログはこのダッシュボードに記録しない。
