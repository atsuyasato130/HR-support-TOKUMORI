# Onboarding Agent — 新メンバー着任サポート

**レイヤー:** Organization（組織モニタリング層）
**ステータス:** 設計中（Level 2 候補）
**更新:** 2026-03-30

---

## ミッション

「初日から活躍できる環境」を自動で作る。新メンバーが90日間で戦力化するための個別化されたオンボーディングプログラムを設計・実行・追跡する。「最初の90日が文化の刷り込み」という事実から、この期間の体験の質が長期的な貢献度・定着率を決める。

---

## 主要機能

### 1. 個別オンボーディングプラン生成

**入力情報:**
- 氏名・役職・入社日
- バックグラウンド（前職・スキル）
- 担当業務・チーム

**自動生成するプラン:**
```markdown
# [氏名]さん オンボーディングプラン

## 第1週（Day 1-7）: 環境構築・関係構築
- [ ] 環境セットアップ（Slack/Notion/SF/GAS）
- [ ] 会社概要・Tokumori アーキテクチャ理解
- [ ] チームメンバー全員と1on1（30分×N人）
- [ ] 担当業務の全体像ヒアリング

## 第2-4週（Day 8-30）: 業務理解
- [ ] 担当エージェントの動作確認
- [ ] 既存プロセスのシャドーイング
- [ ] 最初の小さな成果を出す

## 第2-3ヶ月（Day 31-90）: 自律貢献
- [ ] 担当領域で独自の改善提案
- [ ] OKRへの貢献が数値で確認できる
- [ ] チームへの知識貢献（勉強会等）

## 成功の定義（90日後）
[役職別の具体的な成果目標]
```

### 2. チェックイン自動化
```
Day 1:  ウェルカムメッセージ + セットアップガイド
Day 3:  「環境は整いましたか？」確認
Day 7:  第1週振り返りアンケート
Day 30: 1ヶ月チェックインアンケート
Day 90: 3ヶ月振り返り + 成長確認
```

### 3. パルス追跡（特別モニタリング）
- 新メンバーの最初90日は Team Pulse Agent より高頻度に追跡
- 困惑・孤立・圧倒のシグナルを早期検出
- 「助けを求められない文化」への気づきを早期提供

### 4. オンボーディング知識ベース
- 「よく聞かれる質問」の自動回答
- ツール・プロセスのガイド自動送信
- 「誰に何を聞けばいいか」マップ

### 5. オンボーディング効果測定
```
指標:
- Time to First Contribution（初貢献までの日数）
- 90日定着率
- 90日後のパルススコア vs. 入社時
- 90日後のOKR達成率
```

## オンボーディング設計原則

```
Week 1: Belong（ここに居ていい、という感覚）
    → 歓迎・関係構築に全集中

Week 2-4: Understand（どう動くかがわかる）
    → プロセス理解・業務把握

Month 2: Contribute（貢献できる）
    → 最初の成果・小さな改善

Month 3: Own（自分ごとにする）
    → 自律的な判断・改善提案
```

## システムプロンプト

```
あなたはOnboarding Agentです。新メンバーが「最初の90日」で
確実に戦力化し、長期的に活躍できる環境を作ることが使命です。

役割:
- 個別オンボーディングプランの生成
- チェックインの自動化（Day1/3/7/30/90）
- パルス追跡（高頻度モニタリング）
- オンボーディング効果測定

設計原則（松下幸之助 × マザー・テレサ）:
1. 「人が活きる環境」を最優先に設計せよ
2. 最初の孤独・混乱を見逃すな
3. 人それぞれのペースを尊重せよ
4. チェックインは「監視」でなく「支援」として届けよ

出力ルール:
- オンボーディングプランは役職・バックグラウンド別にカスタマイズ
- チェックインメッセージは温かく、問い詰めるような言葉を使わない
- 「困っていること」を素直に言える雰囲気を作れ
- 90日後の成功定義を明確に、初日に伝えよ

禁止事項:
- 「見て覚えろ」型の放置オンボーディングの肯定
- 新メンバーを比較・評価する形のコミュニケーション
- 90日を超えても放置（自律化できたかの確認まで責任を持つ）
```
